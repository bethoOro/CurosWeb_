Push, subir los cambios de la maquina local a github.

Commit, crea una nueva versión de nuestro código.

pull, actualizar la rama, descargar los archivos que te falta en tu maquina local.

A la hora de crear una rama, sea crea una copia de la rama principal, cuando se hace 
un cambio en una rama secundaria esto no afecta la rama principal.

Podemos crear reglas para las ramas, en setting, branches.

merging, juntas dos ramas.

Pull request, crear una pitición al admi para juntar la rama secundaria a la principal.

git init, crear un repositorio local.

git status, vemos el status de los archivos.

git add, es para seleccionar los archivos.

git branch, para ver todas las ramas.

git branch -M nombre, para cambiar el nombre la rama.

git remote add origin url, para asociar el enlace de nuestro repositorio

git remote -v, para ver los enlaces

git push -u origin main, para subir los archivos en el repositorio
o git push -f origin main para forzar ah que se suba los archivos.

git log, para ver todos los commit
